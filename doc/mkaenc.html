mka encoder


mka encoder version 1.0.0 – utility to encode a wav file + a cue file to a single mka file.

Copyright (C) 2004 Nicolas "goldenear" Le Guen



Presentation:


This software interoperates with a ripping tool (especially Exact Audio Copy) and acts as an external encoder. It permits to directly generate a singe .mka (matroska audio) file backup of a whole audio CD. To do so, it will successively process several operations:

  1. Get file name, tags and parameters informations from the command line generated by the CD ripping software. For example, EAC will run mkaenc.exe with the following parameters : -i "%s" -s "%o" -y "%Y" -g "%m" -m -f

  2. Modify the .cue file (with -m) adding missing parameters: Some informations such as DATE or GENRE are not included in the .cue file generated by the ripping tool, so mkaenc will automatically add this informations (get with -y “%Y” -g”%m”)

  3. Rename the .wav file and the .cue file according to the naming scheme (default -n “/a - /t” will produce files artist – title.wav and artist – title.cue)

  4. Encode the .wav file with an external (preferably lossless) encoder (default is TTA)

  5. delete the .wav file when encoded is completed (unless you use the -w option)

  6. merge the encoded file to a matroska file, use the .cue file to tag chapters and attach this .cue file to the .mka file (unless you use -b)

  7. delete the encoded file and the cue file (unless you use -k)

  8. it's done!


Options and command line parameters:


Required parameters:


-i <input file> Give the wav file to encode with its full path. The cue file must be at the same place.


Optional parameters:


-s <filename> Give the source file name from EAC: You must use “-s %o”


-n <naming scheme> This is how you want the files to be named or renamed. Your can use the special commands:

/a = CD title /t = CD title /y = CD date /g =CD genre /d = DiscID /c = Comment

It uses the tags given by -a -t -y -g -d or -c (see after)

You can also specify a path. For example, -n “D:\audio\/a\/a - /t” will create the files “d:\audio\artist\artist – title.*”

The default is -n “/a - /t”. if you use the -u option, -n will be ignored.


-p <parameters> Optional parameters for mkvmerge.exe


-r <command> This will specify a command or a script to launch after everything is done.


-e <encoder with full path> Give the encoder executable with its full path. The default is "C:\Program Files\TTA\ttaenc.exe". -e can also accept the short name of the encoder, i.e. “FLAC”, “TTA”, of “WAV”. This will set the default parameters for the specified encoder:

FLAC --> executable “C:\Program Files\FLAC\flac.exe”, options “--best -V”, extension “.flac”

TTA --> executable “C:\Program Files\TTA\ttaenc.exe”, options “-e”, extension “.tta”

WAV --> use the wav file directly, do not encode.


-o <encoder options> Specify the options given to the encoder. By default "--best -V"


-x <extension> Specify the encoded file extension. By default ".flac"




Tags parameters:


-a <album artist> Album artist. If not specified, the information if taken from the CUE sheet.


-t <album title> Album title. If not specified, the information if taken from the CUE sheet.


-y <album date> set DATE tag


-g <album genre> set GENRE tag


-d <discID> set DISCID tag


-c <comment> set COMMENT tag




General options:


-m Use this to modify the CUE file. This will add missing tags (for example GENRE or DATE) to the CUE sheet.


-f Fix EAC CUE file extension. EAC creates a cue file with a .mka.cue extension. This option permit to give only the “.cue” extension.


-j Join the CUE file as an attachment to the mka file.


-u Unchange base file name. If you use this option, the files wont be renamed. This cancels the -n option.


-w Don't delete the WAV file after encoding. The cue file won't be deleted after merging.


-k Don't delete the encoded file and the cue file after merging.


-b Don't merge the CUE and encoded files. Use this if you only want to encode and modify the cue file.


-q Quiet mode. Don't display anything except error messages. Encoder messages will still be displayed.


-h Display the full help.


-v Display the software version.


-l Display licence informations.
